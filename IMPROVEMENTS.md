# Список покращань для проєкту Секундомір

## Покращання продуктивності та архітектури

1. **Збереження стану в localStorage** ✅
   - Зберігайте стан таймера, коли користувач закриває вкладку
   - Відновлюйте прогрес при поверненні
   - Це запобігає втраті даних при випадковому закритті

2. **Рефакторинг логіки таймера** ✅
   - Винесіть логіку таймера в окремий Svelte store або composable
   - Це зробить код більш тестованим та повторно використовуваним

3. **Виправлення parseTime** ✅
   - Функція `parseTime` в src/lib/index.ts:22 не обробляє формат з годинами
   - Це може призвести до помилок при довгих сесіях

4. **Оптимізація оновлення часу**
   - Інтервал 10ms може бути надмірним для браузера
   - Розгляньте використання `requestAnimationFrame` для більш плавного рендерингу

## UX покращання

5. **Підтвердження скидання**
   - Додайте діалог підтвердження перед скиданням таймера
   - Запобігає випадковій втраті даних

6. **Експорт результатів**
   - Додайте можливість експортувати результати у CSV/JSON
   - Корисно для аналізу тренувань

7. **Звукові сигнали**
   - Додайте опціональний звуковий сигнал при завершенні кола
   - Покращить досвід використання без постійного перегляду екрану

8. **Статистика в реальному часі**
   - Показуйте поточну швидкість (не тільки середню)
   - Додайте графік швидкості по колах

## Технічні покращання

9. **Типізація подій**
   - Додайте більш строгу типізацію для обробників подій

10. **Доступність (a11y)**
    - Додайте ARIA-labels для кнопок з іконками
    - Додайте role="status" для таймера
    - Покращить досвід для користувачів скрін-рідерів

11. **Responsive design**
    - Оптимізуйте для мобільних пристроїв
    - Sidebar може бути drawer знизу на мобільних

12. **PWA можливості**
    - Додайте service worker для офлайн роботи
    - Додайте manifest.json для встановлення як додаток

## Виправлення помилок

13. **Очищення interval** ✅
    - Додайте `onDestroy` hook для очищення інтервалу
    - Запобігає витокам пам'яті

14. **Обробка edge cases**
    - Що відбувається при швидкому подвійному кліку на "Коло"?
    - Додайте debounce або disable кнопки на момент обробки
